计算机网络-应用层

发表于2022-10-04|更新于2023-07-04|计算机网络
|阅读量:

第二章

应用层原理

传输层向应用层提供的服务：socket API（socket 套接字）

网络应用程序体系结构

C/S模式

  • 客户端IP不固定

  • 资源在服务器，服务器IP固定

  • 可扩展性差，只能从服务器周围扩展。用户数量上升后，越过阈值性能急剧下降。

（但这仍是最常见的）

P2P系统（Peer to Peer）

  • 请求的同时，也接受

  • 每一个节点即是客户端又是服务器

  • 难以管理

也有混合使用的。

进程通信

同一台设备上的进程可以通过操作系统自己的管道通信。不同设备的进程需要依靠计算机网络。

客户与服务器进程如何区分：发起通信的进程识别为客户，在会话开始前等待联系的进程是服务器。

进程与计算机网络之间的接口：socket

分布式需要解决的问题

问题一：进程标识（标识自己）、寻址（怎么找到我）

问题二：传输层-应用层提供的服务是怎么样的

问题三：如何使用传输层提供的服务，实现应用进程之间的报文交换，实现应用

问题一：标识

你在哪（IP地址）？是TCP还是UDP？在TCP上的哪个端口？

主机上有很多进程，用端口号来区分。

一个进程：用IP+port标示端节点

本质上，一对主机进程之间的通信由2个端节点构成

问题二：传输需要携带的信息

首先，SDU肯定要传。然后，谁传出的信息。最后，传给谁的信息。

类比为货物，发件人信息，收件人信息。

传输层实体（TCP或者UDP实体）根据这些信息进行TCP报文段（UDP数据报）的封装

衍生的问题

如果长时间通信，Socket API 每次传输报文，都携带如此多的信息，太繁琐易错，不便于管理。

对方给我返回一个整数，可以用一个整数表示两个应用实体之间的通信关系，本地标示

四元组:源端系统ip、源端系统port、目标端系统ip、目标端系统port

源主机ip:告诉目标主机我是网络上的哪台主机

源主机port:告诉目标主机我是源主机的哪个进程

目标主机ip:我客户端要通信的服务器主机在哪里

目标主机port:我客户端要通信的服务器是哪个进程

所以四元组决定了socket，唯一的TCP socket对应唯一的通信

UDP也有socket，但是只是包含目标端系统ip、目标端系统port的二元组

然而，两者都没有安全性，没有加密机制，TCP有了加强版本。SSL

Web与HTTP

基本概念

HTTP：HyperText Transfer Protocol，超文本传输协议

HTML是超文本标记语言，超文本可以简单的理解为蓝色的链接。我们可以通过一个又一个的链接跳转到不同的网页。这就像一张网一样（Web）

URL：Universal Resource Locator，统一资源标识符。

持续连接

HTTP1.0非持久，每次传输都要建立TCP，会浪费很多资源，因为建立TCP要经过三次握手，每次都重新建立的话会浪费很多时间。

HTTP1.1持久，建立TCP之后，会保持一段时间，这段时间里面可以持续传输数据。

三次握手与传输时间

先讲解一下TCP三次握手，客户端发送建立TCP连接的请求（第一次），服务器收到之后，返回收到的信息（第二次）。客户端收到服务器的收到之后，发送数据（第三次）。

往返时间RTT（round-trip time）：一个小的分组从客户端到服务器，在回到客户端的时间（传输时间忽略）

响应时间：
一个RTT用来发起TCP连接
一个RTT用来HTTP请求并等待HTTP响应
文件传输时间
共：2RTT+传输时间

流水线型：持续发送

Cookie

HTTP无状态协议，需要记录则使用cookie

4个组成部分：

  1. 在HTTP响应报文中有一个cookie的首部行
  2. 在HTTP请求报文含有一个cookie的首部行
  3. 在用户端系统中保留有一个cookie文件，由用户的浏览器管理
  4. 在Web站点有一个后端数据库Web缓存

web 缓存

缓存既是客户端又是服务器
通常缓存是由ISP安装(大学、公司、居民区ISP)

为什么要使用Web缓存？

降低客户端的请求响应时间
可以大大减少一个机构内部网络与Internent接入链路上的流量
互联网大量采用了缓存：可以使较弱的ICP也能够有效提供内容

示例：

，当流量强度接近1时，排队延时趋近于无穷

解决方法一：扩大带宽、降低了流量强度

解决方法二：安排一个缓存节点（很像cache）

带来了新的问题，缓存的内容是否已经陈旧。

所以增加了条件GET方法。在请求报文中包含一个”If-Modified-Since: ”首部行。如果缓存器中的对象拷贝是在这个时间后没有修改过，就不要发送对象。

控制连接负责身份确认、浏览远程目录等等

DNS

人类世界中每个人都有名字和身份证号，人类之间交流常常采用名字，但是为了精确无误的找到某个人，还是需要身份证号才行，但是我们人类不喜欢记忆身份证号，而是喜欢记忆名字。

在互联网上，我们记一个域名比记一个IP地址会好得多。

DNS的功能就是将主机名翻译成IP地址。（Domain Name System），注意哦，DNS也是应用层协议。

DNS运行在UDP之上。而用户之后与真正的目标主机之间建立的连接则往往是TCP连接。

主机别名

多个名字对应同一个IP地址，一个主机名字也可以对应多个IP地址。

负载分配

繁忙的站点（网站）会被冗余分布到多台服务器上，用户的DNS请求会得到IP地址的集合，并且循环的依次与每个服务器进行交流。

工作机理

DNS服务器会在”附近的“DNS服务器中增设缓存，用户可以先在附件的DNS服务器中寻找

DNS是分布式的，而且也分了根DNS服务器、顶级域DNS服务器、权威DNS服务器。互联网经典套娃

这个时候有一个问题，我们怎么知道我们该去哪里找DNS服务器呢？这个问题会在网络层：控制平面讲解。现在我们先不关心下层的实现，只需要知道我们是去DNS服务器上找到IP就可以了

P2P文件分发

一个主机在P2P文件分发中，既是客户端，也是服务器。即接收数据，也发送数据。

BitTorrent，比特流。BT种子

传统C-S和P2P的分发时间对比

视频

服务器收到视频后，会生成多个分辨率的副本。客户端请求数据时，服务器一方面根据传输速率决定分辨率，一方面缓存一部分在用户端，使得用户不会感觉到卡顿

CDN

内容分发网络，可以绕开ISP，直接传输。
